Ветка - изозированный поток разработки (еще одна версия проекта)

СТРАТЕГИИ ВЕТВЛЕНИЯ ДЛЯ GIT

(1) GIT FLOW
	- master / main / stable (long-term)
	- develop / current (long-term)
	- issue553, ticket03, hotfix1, feature/payments, login (short-term)
	

(2) GITHUB FLOW
	- long-term branch
	- short-term branches


git branch новая_ветка


git branch (показать список веток)
git branch новая_ветвь (создать ветку)
git checkout название_ветки (переключиться на ветку)


GIT FLOW (порядок работы по проекту)

(1) внести необх. правки
(2) проиндексировать изменения
(3) выполнить сохранение/фиксацию (git commit)


git commit -am 'текст комментария'

	-m (указать коммент)
	-a (проиндексировать изменения в отслеж. файлах)


СЛИЯНИЕ ВЕТВЕЙ В GIT

git merge название_ветки

название_ветки (удалить)
git checkout -b название_ветки (создать и переключиться)


КАК ВЫГРУЗИТЬ РЕП НА GITHUB?

(1) создать новый реп на GH
(2) сделать привязку локального к дистанционному на GH (git remote)
(3) отправить локальный реп на GH (git push)

КАК УДАЛИТЬ ВЕТВЬ

(1) уйти с нее
(2) удалить локально (git branch -D название_ветки)
(3) удалить на GH (git push origin --delete название_ветки)

git remote rm origin  (удалить старую привязку к реп на GH -- при необх. (!))

git commit --amend (перезапись посл. коммита - внимание! деструктивная операция)
